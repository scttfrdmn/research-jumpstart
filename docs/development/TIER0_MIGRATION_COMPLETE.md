## Technical Details

### Zenodo API Integration
All DOIs were minted programmatically using the Zenodo REST API:
1. Create deposit
2. Upload v1.0.0 release tarball
3. Add metadata (title, description, keywords, license)
4. Publish to receive DOI

### Submodule Structure
Each tier-0 project is located at:
```
projects/<domain>/<project-name>/tier-0-submodule/
```

Main repository tracks submodule commits via `.gitmodules`.

### Clone Instructions

**Full platform with all submodules:**
```bash
git clone --recurse-submodules https://github.com/scttfrdmn/research-jumpstart.git
```

**Individual tutorial:**
```bash
git clone https://github.com/scttfrdmn/rj-agriculture-crop-disease-detection-tier0.git
```

## Maintenance Notes

### Updating a Tier-0 Project
1. Make changes in the submodule directory
2. Commit and push in the submodule
3. Create new release (v1.0.1, etc.) in submodule repo
4. Update Zenodo with new version (creates versioned DOI)
5. Update main repo submodule pointer: `git submodule update --remote`

### Adding New Tier-0 Projects
Use the migration scripts as templates:
1. `/tmp/migrate-tier0-project.sh` - Populate GitHub repo
2. `/tmp/mint-zenodo-doi.sh` - Create DOI
3. `/tmp/fix-readme-doi.sh` - Update badge
4. `git submodule add` - Integrate into main repo
